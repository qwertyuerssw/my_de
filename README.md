
# my_de — Модульное окружение рабочего стола на Rust

Добро пожаловать в **my_de**! Это кастомное, высокомодульное и легковесное окружение рабочего стола (Desktop Environment), написанное на Rust. Проект спроектирован так, чтобы каждый компонент был независимым, а расширение функционала было максимально простым благодаря встроенному SDK.

## 🏗 Архитектура

Проект разделен на несколько логических компонентов (crates), каждый из которых выполняет свою специфическую задачу. Все они общаются между собой посредством механизма межпроцессного взаимодействия (IPC).

1. **`de-compositor`** — Оконный менеджер и графический композитор. Отвечает за отрисовку окон, обработку ввода (мышь, клавиатура) и взаимодействие с графическим сервером (Wayland/X11).
2. **`de-manager`** — Менеджер сессий (оркестратор). Центральный узел, который читает конфигурацию из `de-config.json`, запускает композитор, панель и управляет жизненным циклом всех модулей (расширений).
3. **`de-ipc`** — Протокол связи. Предоставляет структуры данных и методы (сериализацию/десериализацию) для обмена сообщениями между менеджером, композитором и сторонними модулями.
4. **`de-panel`** — Пользовательская графическая оболочка (статус-бар / панель задач). Отображает рабочие столы, системный трей, часы и интегрируется с модулями для вывода информации.
5. **`de-sdk`** — Набор инструментов для разработчиков (Software Development Kit). Содержит вспомогательные функции, абстракции и трейты для легкого написания собственных модулей без необходимости вникать в устройство IPC.
6. **`modules/`** — Директория со встроенными базовыми модулями.

## 🚀 Запуск и сборка

Для запуска окружения в репозитории предусмотрен скрипт:

```bash
./run-de.sh

Он автоматически соберет необходимые компоненты через cargo и запустит
de-manager на основе конфигурации de-config.json.

🛠 Как создать своё расширение (модуль)

Благодаря продуманной архитектуре и наличию de-sdk, написание собственного
расширения (например, виджета погоды, кастомного меню или системного
монитора) — это простой и понятный процесс.

Шаг 1: Инициализация проекта

Создайте новый бинарный пакет (crate) для вашего модуля:

cargo new --bin my_custom_module
cd my_custom_module

Шаг 2: Подключение зависимостей

В файле Cargo.toml вашего нового проекта необходимо подключить SDK вашего
окружения и библиотеку для работы с IPC. Укажите относительный путь к
локальным компонентам:

[dependencies]
de-sdk = { path = "../my_de/de-sdk" }
de-ipc = { path = "../my_de/de-ipc" }
serde = { version = "1.0", features = ["derive"] }

Шаг 3: Написание кода модуля

Используйте трейты и абстракции из de-sdk для подключения к IPC и регистрации
модуля в системе. Ниже приведен пример базового модуля:

use de_sdk::module::{Module, Context};
use de_ipc::events::Event;

// 1. Создаем структуру вашего модуля
struct MyCustomModule;

// 2. Реализуем трейт Module из de-sdk
impl Module for MyCustomModule {
    fn new() -> Self {
        MyCustomModule
    }

    // Инициализация (вызывается при запуске менеджером)
    fn on_start(&mut self, ctx: &mut Context) {
        println!("Модуль MyCustomModule успешно запущен!");
        
        // Пример: подписываемся на события изменения рабочих столов
        ctx.subscribe_to("workspace_changed");
    }

    // Обработка входящих событий (от compositor, panel или manager)
    fn on_event(&mut self, event: Event, ctx: &mut Context) {
        match event.name.as_str() {
            "workspace_changed" => {
                println!("Пользователь переключился на другой рабочий стол!");
                // Здесь можно отправить данные в de-panel для обновления UI
            }
            _ => {}
        }
    }
}

// 3. Запуск модуля
fn main() {
    de_sdk::run_module::<MyCustomModule>();
}

Шаг 4: Сборка и установка

Скомпилируйте ваш модуль в релизной сборке:

cargo build --release

Переместите полученный бинарный файл target/release/my_custom_module в
директорию modules/ или любую другую папку, из которой ваше DE
загружает модули.

Шаг 5: Регистрация в конфигурации

Откройте файл de-config.json и добавьте ваш бинарный файл в список автозагрузки,
чтобы de-manager знал о его существовании и запустил вместе с системой.

{
  "modules": [
    "./modules/default_clock",
    "./modules/default_battery",
    "./modules/my_custom_module"
  ]
}

Перезапустите окружение (через ./run-de.sh), и ваш новый модуль начнет работу!
