
# My DE (Modular Desktop Environment)

Проект представляет собой легковесное модульное графическое окружение (Desktop Environment), написанное на языке Rust. Архитектура построена по принципу разделения обязанностей: ключевые компоненты (композитор, менеджер, панель) работают как отдельные процессы и взаимодействуют друг с другом через шину межпроцессного взаимодействия (IPC).

---

## Архитектура проекта и назначение компонентов

Проект организован в виде Cargo-воркспейса. Ниже приведено описание основных каталогов и их зон ответственности:

### 1. `de-compositor`
**Композитор** — ядро графического окружения. Он выполняет роль дисплейного сервера (вероятнее всего, Wayland-композитора) и отвечает за:
* Отрисовку и композицию окон (поверхностей).
* Обработку устройств ввода (клавиатура, мышь, тачпад).
* Вывод графической информации на экраны (мониторы).
* Управление базовыми состояниями оконных поверхностей.

### 2. `de-ipc` (Inter-Process Communication)
**Модуль межпроцессного взаимодействия** — библиотека, обеспечивающая протокол общения между независимыми процессами окружения. Поскольку композитор, панель и менеджер запускаются отдельно, `de-ipc` отвечает за:
* Передачу системных событий (например, изменение активного окна, переключение воркспейса).
* Отправку команд управления (например, закрытие окна, запуск приложения).
* Синхронизацию конфигурации между модулями в реальном времени.

### 3. `de-manager`
**Менеджер сессии / окон** — управляющий компонент. Он координирует работу всего окружения:
* Считывает настройки из конфигурационного файла `de-config.json`.
* Управляет жизненным циклом других компонентов (запуск, перезапуск в случае сбоя).
* Применяет глобальные правила поведения окон и рабочих пространств.

### 4. `de-panel`
**Панель управления (статус-бар)** — графический интерфейс для пользователя. Обычно располагается на краю экрана и отображает:
* Список активных рабочих столов (Workspaces).
* Запущенные или закрепленные приложения.
* Системную информацию (часы, статус сети, уровень заряда батареи, системный трей).
* Динамически обновляет данные, получая информацию от других компонентов через `de-ipc`.

### 5. `modules`
**Вспомогательные модули** — каталог, содержащий разделяемые библиотеки, общие структуры данных, утилиты логирования или плагины, которые могут использоваться одновременно несколькими компонентами воркспейса для исключения дублирования кода.

---

## Основные конфигурационные и системные файлы

* **`de-config.json`** — файл конфигурации в формате JSON. В нем определяются параметры окружения, такие как настройки панели, горячие клавиши (keybindings), правила автозапуска и другие предпочтения пользователя.
* **`run-de.sh`** — shell-скрипт для инициализации и запуска окружения. Он настраивает необходимые переменные окружения (например, переменные Wayland/X11), запускает композитор, а затем последовательно инициализирует менеджер и панель.
* **`Cargo.toml`** — манифест Cargo, объявляющий структуру воркспейса и общие зависимости для всех подпроектов.
* **`compositor.log`** — лог-файл, в который записывается вывод процесса композитора для упрощения отладки графического сервера.

              |    run-de.sh     |  <-- Скрипт инициализации
              +--------+---------+
                       |
                       v
              +------------------+
              |    de-manager    |  <-- Читает de-config.json
              +--------+---------+
                       |
     +-----------------+-----------------+
     | (IPC)                             | (IPC)
     v                                   v

    +--------+---------+ +---------+--------+ 
    | de-compositor   | |de-panel | | | | |
    | (Вывод/Графика) | | (Интерфейс/Бар) |

## Сборка и запуск

### Требования
Для сборки проекта вам понадобятся:
* Rust toolchain (cargo, rustc версии, поддерживающей ваш набор зависимостей).
* Системные библиотеки для работы с графикой (например, библиотеки Wayland/X11, xkbcommon, в зависимости от используемых под капотом библиотек отрисовки).

### Сборка
Для компиляции всех модулей воркспейса выполните команду из корневой директории:
```bash
cargo build --release

Запуск

Для тестирования или полноценного запуска окружения используйте стартовый
скрипт:

./run-de.sh
