
# My DE (Modular Desktop Environment)

Проект представляет собой легковесное модульное графическое окружение (Desktop Environment), написанное на языке Rust. Архитектура построена по принципу разделения обязанностей: ключевые компоненты (композитор, менеджер, панель) работают как отдельные процессы и взаимодействуют друг с другом через шину межпроцессного взаимодействия (IPC).

---

## Архитектура проекта и назначение компонентов

Проект организован в виде Cargo-воркспейса. Ниже приведено описание основных каталогов и их зон ответственности:

### 1. `de-compositor`
**Композитор** — ядро графического окружения. Он выполняет роль дисплейного сервера (вероятнее всего, Wayland-композитора) и отвечает за:
* Отрисовку и композицию окон (поверхностей).
* Обработку устройств ввода (клавиатура, мышь, тачпад).
* Вывод графической информации на экраны (мониторы).
* Управление базовыми состояниями оконных поверхностей.

### 2. `de-ipc` (Inter-Process Communication)
**Модуль межпроцессного взаимодействия** — библиотека, обеспечивающая протокол общения между независимыми процессами окружения. Поскольку композитор, панель и менеджер запускаются отдельно, `de-ipc` отвечает за:
* Передачу системных событий (например, изменение активного окна, переключение воркспейса).
* Отправку команд управления (например, закрытие окна, запуск приложения).
* Синхронизацию конфигурации между модулями в реальном времени.

### 3. `de-manager`
**Менеджер сессии / окон** — управляющий компонент. Он координирует работу всего окружения:
* Считывает настройки из конфигурационного файла `de-config.json`.
* Управляет жизненным циклом других компонентов (запуск, перезапуск в случае сбоя).
* Применяет глобальные правила поведения окон и рабочих пространств.

### 4. `de-panel`
**Панель управления (статус-бар)** — графический интерфейс для пользователя. Обычно располагается на краю экрана и отображает:
* Список активных рабочих столов (Workspaces).
* Запущенные или закрепленные приложения.
* Системную информацию (часы, статус сети, уровень заряда батареи, системный трей).
* Динамически обновляет данные, получая информацию от других компонентов через `de-ipc`.

### 5. `modules`
**Вспомогательные модули** — каталог, содержащий разделяемые библиотеки, общие структуры данных, утилиты логирования или плагины, которые могут использоваться одновременно несколькими компонентами воркспейса для исключения дублирования кода.

---

## Основные конфигурационные и системные файлы

* **`de-config.json`** — файл конфигурации в формате JSON. В нем определяются параметры окружения, такие как настройки панели, горячие клавиши (keybindings), правила автозапуска и другие предпочтения пользователя.
* **`run-de.sh`** — shell-скрипт для инициализации и запуска окружения. Он настраивает необходимые переменные окружения (например, переменные Wayland/X11), запускает композитор, а затем последовательно инициализирует менеджер и панель.
* **`Cargo.toml`** — манифест Cargo, объявляющий структуру воркспейса и общие зависимости для всех подпроектов.
* **`compositor.log`** — лог-файл, в который записывается вывод процесса композитора для упрощения отладки графического сервера.

## Сборка и запуск

### Требования
Для сборки проекта вам понадобятся:
* Rust toolchain (cargo, rustc версии, поддерживающей ваш набор зависимостей).
* Системные библиотеки для работы с графикой (например, библиотеки Wayland/X11, xkbcommon, в зависимости от используемых под капотом библиотек отрисовки).

### Сборка
Для компиляции всех модулей воркспейса выполните команду из корневой директории:
```bash
cargo build --release

Запуск

Для тестирования или полноценного запуска окружения используйте стартовый
скрипт:

./run-de.sh

```

Благодаря архитектуре со слабой связанностью (decoupling) компонентов,
добавление новой функциональности происходит без вмешательства в
сложный графический код композитора.

Инструкция по созданию и интеграции нового модуля в MyDE (на примере модуля звука)

Процесс добавления нового модуля состоит из 5 основных этапов:

1.  Расширение общего протокола взаимодействия (de-ipc).
2.  Создание нового проекта-крейта для модуля и его регистрация в воркспейсе.
3.  Обучение диспетчера (de-manager) управлению новым процессом.
4.  Добавление поддержки модуля и элементов интерфейса в панель (de-panel).
5.  Сборка и тестирование.

Шаг 1: Расширение протокола в de-ipc

Чтобы все компоненты системы научились распознавать новый модуль, необходимо
добавить его уникальный идентификатор в общее перечисление.

1.  Откройте файл de-ipc/src/lib.rs.
2.  Добавьте вариант Volume в перечисление ModuleId:

#[derive(Debug, Serialize, Deserialize, Clone, Copy, PartialEq, Eq, Hash)]
pub enum ModuleId {
    Clock,
    SysInfo,
    Volume, // <--- Новый идентификатор модуля звука
}

Шаг 2: Создание крейта модуля и настройка Workspace

1. Регистрация в корневом воркспейсе (Критически важный шаг)

Чтобы Cargo знал о существовании нового модуля и компилировал его вместе со всем
проектом, откройте корневой файл Cargo.toml в корневой папке проекта
(my_de/Cargo.toml) и добавьте путь к новому модулю в секцию members:

[workspace]
resolver = "2"
members = [
    "de-compositor",
    "de-ipc",
    "de-manager",
    "de-panel",
    "modules/mod-clock",
    "modules/mod-sysinfo",
    "modules/mod-volume"  # <--- Обязательно добавьте этот путь
]

2. Создание структуры каталогов модуля

Создайте директорию для нового модуля по пути: modules/mod-volume/. Внутри неё
создайте два файла: Cargo.toml и src/main.rs.

3. Настройка modules/mod-volume/Cargo.toml

Определите зависимости модуля. Ему понадобятся библиотека протокола de-ipc и
библиотеки сериализации:

[package]
name = "mod-volume"
version = "0.1.0"
edition = "2021"

[dependencies]
de-ipc = { path = "../../de-ipc" }
serde = { version = "1.0", features = ["derive"] }
serde_json = "1.0"

4. Написание кода в modules/mod-volume/src/main.rs

Код модуля должен подключаться к Unix-сокету IPC, регистрироваться как клиент
типа Volume и в цикле передавать данные. Для примера ниже приведена симуляция
изменения громкости (в реальном модуле здесь может использоваться вызов утилит
вроде amixer или библиотек ALSA/PulseAudio/PipeWire):

use std::io::Write;
use std::os::unix::net::UnixStream;
use std::thread;
use std::time::Duration;
use de_ipc::{IpcMessage, ClientType, ModuleId};

fn main() -> Result<(), Box<dyn std::error::Error>> {
    let socket_path = "/tmp/my-de-ipc.sock";
    let mut socket = UnixStream::connect(socket_path)?;

    // Шаг 1: Регистрируемся в системе IPC под типом Volume
    let reg_msg = IpcMessage::Register {
        client_type: ClientType::Module(ModuleId::Volume),
    };
    send_ipc(&mut socket, &reg_msg)?;

    let mut mock_volume = 50; // Стартовое значение для симуляции
    
    loop {
        // Форматируем строку для отправки на панель
        let formatted_data = format!("🔊 Vol: {}%", mock_volume);

        // Формируем сообщение обновления
        let update_msg = IpcMessage::PublishUpdate {
            module: ModuleId::Volume,
            data: formatted_data,
        };

        send_ipc(&mut socket, &update_msg)?;

        // Симулируем изменение громкости для наглядности теста
        mock_volume = (mock_volume + 5) % 100;

        thread::sleep(Duration::from_secs(3));
    }
}

fn send_ipc(socket: &mut UnixStream, msg: &IpcMessage) -> Result<(), Box<dyn std::error::Error>> {
    let mut serialized = serde_json::to_string(msg)?;
    serialized.push('\n');
    socket.write_all(serialized.as_bytes())?;
    socket.flush()?;
    Ok(())
}

Шаг 3: Регистрация в диспетчере процессов de-manager

Диспетчер процессов должен знать, какой бинарный файл запускать и контролировать
при получении команд для ModuleId::Volume.

1.  Откройте файл de-manager/src/main.rs.
2.  В функции main найдите блок инициализации системных модулей в state.modules
    и добавьте туда структуру ModuleProcess для нового модуля:

// ... Инициализация Clock и SysInfo ...

state.modules.insert(
    ModuleId::Volume,
    ModuleProcess {
        id: ModuleId::Volume,
        path: "./target/debug/mod-volume", // Относительный путь к собранному бинарнику
        child: None,
        enabled: false,
    },
);

3.  Откройте конфигурационный файл de-config.json в корне вашего проекта и
    добавьте запись о новом модуле в массив "modules":

{
  "modules": [
    { "id": "Clock", "path": "./target/debug/mod-clock" },
    { "id": "SysInfo", "path": "./target/debug/mod-sysinfo" },
    { "id": "Volume", "path": "./target/debug/mod-volume" } // <--- Добавили конфигурацию пути
  ]
}

Шаг 4: Обновление интерфейса в de-panel

Чтобы отображать состояние громкости и управлять работой модуля с панели,
необходимо обновить логику de-panel/src/main.rs.

1. Обновление полей состояния приложения

Добавьте поля состояния для отслеживания статуса работы модуля (volume_running)
и хранения текущего текстового значения громкости (volume_data) в структуру
PanelApp:

struct PanelApp {
    ipc_tx: std::sync::mpsc::Sender<IpcMessage>,
    ipc_rx: std::sync::mpsc::Receiver<IpcMessage>,
    
    clock_running: bool,
    sysinfo_running: bool,
    volume_running: bool, // <--- Добавили статус работы
    
    clock_data: String,
    sysinfo_data: String,
    volume_data: String,  // <--- Добавили данные громкости
}

2. Обновление инициализации в PanelApp::new

Инициализируйте новые поля начальными значениями внутри конструктора
PanelApp::new. При этом изменять замыкание в функции main не требуется: eframe
автоматически передает контекст cc и возвращает обновленный Self:

impl PanelApp {
    fn new(cc: &eframe::CreationContext<'_>) -> Self {
        // ... (весь ваш прошлый код создания каналов и сокетов остается без изменений) ...

        Self {
            ipc_tx: gui_tx,
            ipc_rx: rx,
            clock_running: false,
            sysinfo_running: false,
            volume_running: false, // <--- Изначально выключен
            clock_data: "Waiting clock data...".to_string(),
            sysinfo_data: "Waiting sysinfo data...".to_string(),
            volume_data: "Waiting volume data...".to_string(), // <--- Инициализация заглушкой
        }
    }
}

3. Обработка IPC-сообщений в цикле обновлений

Внутри метода update (в блоке разбора входящих сообщений ipc_rx) научите
приложение реагировать на сообщения от нового модуля:

// Обработка изменения статуса работы (запущен/остановлен)
IpcMessage::ModuleStatus { module, is_running } => {
    match module {
        ModuleId::Clock => self.clock_running = is_running,
        ModuleId::SysInfo => self.sysinfo_running = is_running,
        ModuleId::Volume => self.volume_running = is_running, // <--- Сюда
    }
}

// Обработка данных обновления
IpcMessage::PublishUpdate { module, data } => {
    match module {
        ModuleId::Clock => self.clock_data = data,
        ModuleId::SysInfo => self.sysinfo_data = data,
        ModuleId::Volume => self.volume_data = data, // <--- И сюда
    }
}

4. Добавление виджета на панель (egui)

В этом же методе update, в блоке отрисовки интерфейса добавьте текстовую метку и
кнопку управления:

// Отрисовка текстового виджета громкости
if self.volume_running {
    ui.label(egui::RichText::new(&self.volume_data).strong().color(egui::Color32::GOLD));
} else {
    ui.label("🔊 Vol: Off");
}

ui.separator();

// Кнопка для запуска/останова модуля в правой части панели
let volume_text = if self.volume_running { "Stop Vol" } else { "Start Vol" };
if ui.button(volume_text).clicked() {
    let action = if self.volume_running { ProcessAction::Stop } else { ProcessAction::Start };
    let _ = self.ipc_tx.send(IpcMessage::ControlModule {
        module: ModuleId::Volume,
        action,
    });
}

Шаг 5: Компиляция и запуск

1.  Сборку и запуск необходимо осуществлять строго из корневой директории
    воркспейса my_de, иначе относительные пути запуска бинарных файлов
    (указанные в de-manager и de-config.json) могут не разрешиться корректно.
2.  Соберите проект:
    cargo build
3.  Убедитесь, что исполняемый файл был успешно скомпилирован и находится по
    адресу target/debug/mod-volume.
4.  Запустите окружение с помощью скрипта автоматического развертывания:
    ./run-de.sh

После запуска на панели отобразится новая кнопка управления звуком, при нажатии
на которую менеджер безопасно запустит процесс mod-volume, а панель начнет
выводить симулируемые им данные.
